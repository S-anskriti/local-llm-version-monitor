# Local LLM Enterprise Assistant (Proof of Concept)

## Overview

This repository is the starting point of an ongoing exploration into building a **Local Large Language Model (LLM) assistant for enterprise software environments**.

Many organizations rely on cloud-based AI services to analyze software packages, check for updates, identify vulnerabilities, and provide developers with recommendations directly inside their development environments. While these services are powerful, they require sending information to external servers, which may not always align with an organization's privacy, security, or compliance requirements.

The goal of this project is to investigate how a **Local LLM**, running entirely on an organization's own infrastructure, can perform similar tasks while ensuring that sensitive application and software information never leaves the local environment.

Instead of depending on cloud-hosted AI models, this project is designed around **Gemma3:4B running locally using Ollama**, allowing all AI-powered analysis and responses to be generated on the user's machine.

This repository currently contains the initial proof-of-concept demonstrations that explore the core building blocks required for such a system.

---

## Project Vision

The long-term objective is to build a Local LLM-powered enterprise assistant capable of:

* Monitoring installed software and package versions.
* Detecting available software updates.
* Providing natural language explanations for updates.
* Summarizing release notes.
* Highlighting important security updates.
* Integrating directly into developer workflows (such as Visual Studio Code).
* Performing all processing locally without relying on cloud-based AI services.

The current implementation focuses only on the first step of this vision: detecting software version changes and presenting update information.

# Demo 1 – HTML Version Monitor

## Objective

The first demonstration introduces the core version comparison workflow using a simple web interface.

The purpose of this demo is to simulate how an enterprise application could determine whether installed software is up to date before introducing Local LLM capabilities.

## Workflow

1. Read the installed application versions from `installed_apps.json`.
2. Read the latest available versions from `latest_versions.json`, which acts as a simulated external version source.
3. Compare the installed version with the latest available version.
4. Detect whether an update is available.
5. Display update information through a simple HTML interface.

This demonstration focuses on the version comparison logic and establishes the foundation for future Local LLM integration.

## Components

* Python
* JSON
* HTML
* CSS

## Current Features

* Reads installed application versions.
* Reads latest available versions from a simulated external source.
* Compares software versions.
* Detects available updates.
* Displays update information through a web interface.

## Future Enhancement

In future versions, the update notification will be generated by **Gemma3:4B running locally through Ollama**, allowing the application to produce intelligent, human-readable explanations without using any cloud-based AI service.
